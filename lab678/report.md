# Отчёт по лабораторным работам 6-8

## Тема

Построение RAG-системы на основе PostgreSQL, pgvector и локальной LLM.

## Цель

Освоить создание Retrieval-Augmented Generation системы, которая объединяет:

- PostgreSQL 16 и расширение pgvector для хранения векторов;
- модель эмбеддингов sentence-transformers для семантического поиска;
- локальную LLM через Ollama или LM Studio;
- веб-интерфейс для запросов и пополнения базы знаний.

## Ход работы

1. Настроена база данных PostgreSQL с расширением `vector`.
2. Создана таблица `documents` с полями `content`, `embedding`, `metadata`, `created_at`.
3. Добавлен HNSW-индекс `documents_embedding_hnsw_idx` для ускорения векторного поиска.
4. Подключена модель `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`.
5. Реализована индексация документов: текст преобразуется в embedding и сохраняется в PostgreSQL.
6. Реализован RAG-запрос: вопрос преобразуется в embedding, pgvector ищет top-k похожих документов, затем контекст передаётся в локальную LLM.
7. Разработан Streamlit-интерфейс для вопросов, добавления, редактирования, поиска и импорта фактов.
8. Добавлена визуализация релевантности найденных документов и распределения базы знаний по источникам.
9. Добавлена вкладка настроек для изменения `LLM_TIMEOUT`, `LLM_MAX_TOKENS`, `LLM_TEMPERATURE`, `DEFAULT_TOP_K` без ручного редактирования `.env`.

## Контрольные вопросы

### 1. Что такое RAG и какие задачи он решает?

RAG, или Retrieval-Augmented Generation, — это подход, при котором языковая модель отвечает не только на основе внутренних знаний, но и на основе найденных внешних документов. Он решает проблему устаревших или неполных знаний модели, снижает количество выдуманных ответов и позволяет работать с локальной базой знаний.

### 2. Как работает векторный поиск в PostgreSQL с pgvector?

Текст преобразуется в числовой вектор фиксированной размерности. В PostgreSQL расширение pgvector хранит такие векторы в типе `vector` и предоставляет операторы расстояния. В работе используется оператор `<=>`, который считает косинусное расстояние между вектором вопроса и векторами документов. Чем меньше расстояние, тем релевантнее документ.

### 3. Какую роль играет модель эмбеддингов в RAG-системе?

Модель эмбеддингов переводит тексты и вопросы в векторы так, чтобы близкие по смыслу фразы оказывались рядом в векторном пространстве. Благодаря этому можно искать документы не только по совпадению слов, но и по смыслу.

### 4. Почему используется локальная LLM вместо облачных решений?

Локальная LLM позволяет не отправлять данные во внешние сервисы, работать без оплаты облачного API и контролировать окружение эксперимента. Это особенно важно для учебных и корпоративных баз знаний, где могут быть внутренние документы.

### 5. Как можно улучшить качество ответов RAG-системы?

Качество можно улучшить подбором более сильной модели эмбеддингов, увеличением `top_k`, разбиением больших документов на аккуратные фрагменты, фильтрацией по метаданным, добавлением reranker-модели и настройкой промпта для локальной LLM.
